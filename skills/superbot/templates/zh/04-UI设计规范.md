## 信息架构

### 站点地图 / 应用结构

> 以下是一个复杂产品的信息架构示例，展示层级深度、命名范式和上下文锁定关系。实际使用时请替换为你项目的实际内容。

```
[产品名称]（主产品）
│
├── [主模块A]
│   │
│   ├── [功能域A]
│   │   └── [列表页]                                            # 列出该功能域下所有条目和核心指标
│   │       └── [详情页]                                        # 深入理解单个条目的全貌
│   │           ├── [概览页]                                    # 宏观判断整体效果
│   │           ├── [子分析页A]                                 # 识别细分维度的趋势和短板
│   │           │   └── [探索器(锁定功能域A和上下文X)]          # 在上级上下文中逐一排查所有子项
│   │           │       └── [详情视图(锁定功能域A和子分析页A)]  # 同子项多次执行的横向对比
│   │           │           └── [单条记录视图]                  # 逐层还原单次记录全貌，定位问题根因
│   │           ├── [子分析页B]                                 # 评估另一维度的质量与瓶颈
│   │           └── [探索器(锁定功能域A)]                       # 从功能域维度全局查看数据质量与分布
│   │               └── [详情视图]                              # 逐层还原单次记录，定位问题根因
│   │
│   ├── [功能域B]                                               # 管理相关数据资产，掌握数据全景
│   │   └── [列表页]                                            # 列出所有条目
│   │       └── [探索器(锁定功能域B)]                           # 列出特定条目下所有子项及核心指标
│   │           └── [详情页]                                    # 从数据视角分析跨模块、跨场景表现
│   │               └── [探索器(锁定子项)]                      # 查看该子项所有执行记录的原始数据
│   │                   └── [单条记录视图]                      # 逐层还原单次记录，定位问题根因
│   └── ...
│
├── [主模块B]
│   └── ...
└── ...
```
